# CC Switch Integration Guide

This guide walks you through connecting your Agent OS to CC Switch, so your agent can read and write memory files from your IDE.

## Prerequisites

- Windows 10/11
- Python 3.10+ installed and in PATH
- CC Switch installed (Chrome extension or desktop app)
- One of: OpenCode, Codex, or Claude Desktop

---

## Step 1: Run the Setup Script

Open PowerShell in the `Bridge/` folder:

```powershell
cd Bridge
.\setup.ps1
```

The script does three things:
1. Creates `config.json` with your local paths
2. Generates `ccswitch-mcp-config.json` for CC Switch
3. Shows you the next steps

If the script fails, check that Python is installed: `python --version`

---

## Step 2: Add MCP to CC Switch

1. Open CC Switch
2. Click **MCP** at the top
3. Click the **+** button (top right)
4. Choose **Custom**

Fill in these fields:

| Field | Value |
|-------|-------|
| MCP Title (unique) | `agent` |
| Display Name | `Agent OS` |
| Enable for Apps | ☑ OpenCode ☑ Codex ☑ Claude |

Then paste the content of `ccswitch-mcp-config.json` (generated by setup.ps1) into the config area.

---

## Step 3: Add the Bootstrap Prompt

1. In CC Switch, go to your app (OpenCode / Codex / Claude)
2. Click **Prompts**
3. Create a new prompt

| Field | Value |
|-------|-------|
| Name | `Agent OS Bootstrap` |
| Content | Paste the entire content of `Agent-Bootstrap-Prompt.md` |

---

## Step 4: Verify

Restart your CLI, then type:

```
Please call agent_ping first, then tell me the current Agent OS Git status.
```

If the agent responds with Git status info, the integration is working.

---

## How It Works

```
Your IDE ←→ CC Switch ←→ Bridge (Python) ←→ Agent OS files
```

- **agent_wake**: Reads memory files, returns boot context
- **agent_read_memory**: Reads any memory file by path
- **agent_write_memory**: Writes content to a memory file
- **agent_git_status**: Shows Git state of the repo
- **agent_ping**: Health check

---

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "command not found" | Make sure Python is in PATH: `python --version` |
| MCP connection failed | Check CC Switch is running, restart CLI |
| Agent doesn't respond | Verify prompt is pasted correctly in CC Switch |
| Path errors | Edit `config.json` in Bridge folder, set correct paths |

## File Structure After Setup

```
Bridge/
├── config.json              # Your local paths (gitignored)
├── ccswitch-mcp-config.json # Generated for CC Switch (gitignored)
├── server.py                # MCP server
├── core.py                  # Agent OS operations
├── run_mcp.cmd              # launcher
├── Agent-Bootstrap-Prompt.md # Paste this into CC Switch
└── CCSwitch-Guide.md        # This file
```
